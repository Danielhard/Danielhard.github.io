# 目录结构

```
  --src 
    -- compiler
    -- core 
    -- entries
    -- platforms
    -- server
    -- sfc 
    -- shared

```
  1. /copmplier 目录是编译模板
  2. /core目录是vue.js 的核心
  3. /entries目录是生产打包的入口
  4. /platforms目录是针对核心模块的'平台'模块
  5. platforms 目录下暂时只有web目录 （在最新的开发目录下边已经有weex 目录）。web目录下有对应的 /compiler 、 /runtime 、/server 、 /util目录
  6. /server 目录是处理服务端渲染
  7. /sfc目录处理但文件 .vue
  8. /shared 目录提供全局用到的工具函数。
  • 结论： Vue整体组成由core(核心)+对应的'平台'补充代码构成(独立构建和运行时构建只是platforms下web平台的两种选择),

  • Vue.js的目标是通过尽可能简单的API实现响应的数据绑定和组合的视图组件 1.components 模板编译的代码 2.global-api 最上层的文件接口 3.instance生命周期-> init.js 4.observer 数据收集与订阅 5.util常用工具方法类 6.vdom虚拟dom

  • 双向绑定原理 模板渲染解析时watcher会对应绑定指令(一对一).此时会通过调用订阅者watcher初始化(watcher中get()方法)去触发对应属性在发布者observer里(object.defineProperty)的getter,observer会判断是不是通过watcher初始化调用的（Dep.target,实例化之后会清空），只有是才会通过dep类依赖收集，observer通过depend通知Dep类收集(addDep方法，在watcher类中，会传入当前Dep实例调用自身)当前该订阅者(watcher)中的触发更新的方法，同时第一次初始化watcher.update()初始化视图。此后每次的数据更新都会通过observer中的setter去触发dep类中的回调update执行收集依赖的所有方法更新订阅者中的状态同时更新视图。observer在处理对象和数组的时候，如果是数组，并且调用的方法会改变数组长度，则会利用重写原型链方法,执行原始方式，拦截新增，遍历递归代理新增数据，然后手动触发一次setter做的事情(因为调用数组原生APi可能多次触发getter setter 且索引不会变)，如果是对象则 通过对象的getter获取值和setter更新值.

  • observer观察者

       function Dep(){ // 主题对象
      this.subs = []; //订阅者列表
   }
   // 主体对象通知订阅者
   Dep.prototype.notify = function(){
     // 遍历所有的订阅者，执行订阅者提供的更新方法
     this.subs.forEach(function(sub){
       sub.update();
     })
   }

   function Sub(x){ // 订阅者
     this.x = x 
   }
   //订阅者更新 
   Sub.prototype.update = function(){
     this.x = this.x + 1
   }
   

   var pub ={
     publish:function(){
       dep.notify();
     }
   };
   var dep = new Dep(); //主题对象实例
   dep.subs.push(new Sub(1),new Sub(2));
   // 新增2个订阅者
   pub.publish();//发布者发布更新

  1. 创建一个dep ,收集对应的watcher
  2. 对于数据构建响应 同时做了get 和 set处理
  3. defineProperty的get，Dep.target => Dep.addSub =>收集依赖
  4. 定义set => dep.notify
    4.渲染 Complie (返回dom节点插到页面上)
  5. 传入node节点 vm
  6. nodeToFragment => document.createDocumentFragment 创建文档片段 它在内存中，所以不会造成太大的性能问题
  7. while(child = node.firstChild) => complieElement => 添加到文档片段里面
  8. compileElement => 判断节点类型,修正模板 (new Watcher()); 分不同场景获取的值。文本节点的nodevalue 还有指令获取 1 . 文本=> name(test) -> 获取到我们在模板里面使用的data里面的key 2 . 一个指令对应声明一个watcher
  9. watcher实例化 , Dep.target = this => update ->(observer) get ->dep.addSub()
  10. get到值之后，nodeValue = value 同时进行dom-diff get阶段就是打patch的时候。
  11. batcher (vue的调度 eventLoop)

# 3.0

  • Proxy(一层)
  • compiler-core与平台无关的编辑器（AST）
  • compiler-dom专门针对浏览器写的编辑器
  • reactivity 数据相应系统，单独的系统，可以配合任何使用
  • runtime-core 与平台无关的运行时，实现虚拟dom渲染器（虚拟 dom与此层分离 ）、vue组件，vue Api 、可以利用这一层 具体实现某个平台的runtime，自定义渲染器
  • runtime-dom: 专门针对浏览器的运行时，dom api,dom 事件，dom 属性
  • runtime-test:辅助dom渲染测试工具
  • server-renderer : ssr
