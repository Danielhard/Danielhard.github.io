# webpack 模块化打包工具 一切皆模块

# 基础

  • 占位符自定义打包文件名 filename:"[name].[hash:5].js"
  • 什么是loader ? 文件预处理器
  less-loader html-loader css-loader  file-loader 特定的处理方案
  • sourceMap 打包后定位代码文件错误
  // devtool: cheap-module-eval-source-map 开发模式 会暴露源码
  // devtool: cheap-module-source-map 生产环境

  • HMR hot module replacement webpack热更新

    开启步骤 使用webpack-dev-server 作为服务器启动 devServer配置中 hot:true plugins 中 新增 hotModuleReplaceMentPlugin js 模块中增加module.hot.accept 增加HMR代码

  • babel -- js编辑器

    @babel/cli shell可执行 核心: @babel/core

    npm install @babel/core @babel/preset-env babel-loader -D 使用@babel/preset-env ES6+ 语法转换 标准引入语法：箭头函数 let,const 模块等 可以转换

    对于标准引入全局变量。部分原生对象新增的原型链上的方法。不可以(Promise map set Symbol),这时需要用polyfill(垫片)

    @babel/polyfill 全局变量的形式将方法注入，开发类库，UI组件时，全局变量污染。 要和core.js搭配 3.X版本

    @babel/plugin-transform-runtime 以闭包方式注入 保证全局变量环境不被污染 (推荐) 搭配@babel/runtime-corejs3

# 稍微高级点

  • treeShaking 依赖es6模块语法esModule production 模式下会自动压缩，但是配置optimization.inimizer覆盖默认压缩工具，再想压缩js需要用terser-webpack-plugin

# dev和pro环境区分

开发 devServer sourceMap proxy ... 生产 treeShaking 代码压缩 提取公共代码 共同点 同样的入口 部分相同的代码处理

方案 webpack.prod.js webpack.dev.js webpack.base.js 开发环境与生产环境公用的代码 借助一个工具 webpack-merge

# 打包优化

  1. 入口配置：entry多入口 webpack.ProvidePlugin
  2. 抽取公共代码：splitchunksPlugins //webpack4 前commonChunksPlugin
  3. 动态加载：按需(懒)加载 @babel/plugin-syntax-dynamic-import
